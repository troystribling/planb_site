The most notable addition to release 0.1.2 of has_ancestor is support for finders that transparently allow construction of database queries that span the inheritance hierarchy. A new class method, find_by_model, which both has_ancestor and has_descendants add to a model, constructs queries in which conditions may be specified on any attribute in the model inheritance hierarchy. Also new are genearted finders, similar in form to the dynamic finders provided by Rails, but unlike the Rails dynamic finders, which are restricted to attributes belonging to a single model, attributes supported by the has_ancestor generated finders may be supported by any model in the inheritance hierarchy.

A more general discussion of using has_ancestor to construct data models can be found in Construct Model Abstractions With has_ancestor.

Example Models

This section will describe the models that will be used in examples in the following sections. First, the migration for the models is shown followed by the class definitions.

In the class definitions notice that ChildModel has a descendant relation with ParentModel and that GrandchildModel has a descendant relation with ChildModel.

find_by_model

The class method find_by_model will be added to any model that calls either has_ancestor or has_descendants and accepts all options supported by ActiveRecord::Base#find. find_by_model works by automatically generating :joins for the foriegn key references of all ancestors of the class on which the method was called. The generated joins: option along with any other specified options are then passed to ActiveRecord::Base#find. Because of this all options accepted by ActiveRecord::Base#find are also accepted by find_by_model. Since a join over multiple tables is performed, the table supporting the attribute must be specified when the attribute is referenced in options. This is illustrated in following examples of queries that return the first match of a specified value of ParentModel#parent_model_string.

# returns a model of class ParentModel mod = ParentModel.find_by_model(:first, :conditions => "parent_models.parent_model_string = 'value'") mod.class.eql?(ParentModel) # returns true # returns a model of class ChildModel mod = ChildModel.find_by_model(:first, :conditions => "parent_models.parent_model_string = 'value'") mod.class.eql?(ChildModel) # returns true # returns a model of class GrandchildModel mod = GrandchildModel.find_by_model(:first, :conditions => "parent_models.parent_model_string = 'value'") mod.class.eql?(GrandchildModel) # returns true

Note that the references to parent_model_string are of the form parent_models.parent_model_string and that the class of the result of the query matches the class on which the method was called, namely, when called from ParentModel a ParentModel is returned, when called from ChildModel a ChildModel is returned and when called from GrandchildModel a GrandchildModel.

Queries that return all models matching a specified value of parent_models.parent_model_string can be constructed by using the :all option.

# returns a model of class ParentModel mods = ParentModel.find_by_model(:all, :conditions => "parent_models.parent_model_string = 'value'") mods.each {|m| m.class.eql?(ParentModel)} # returns true for each model # returns a model of class ChildModel mods = ChildModel.find_by_model(:all, :conditions => "parent_models.parent_model_string = 'value'") mods.each {|m| m.class.eql?(ChildModel)} # returns true for each model # returns a model of class GrandchildModel mods = GrandchildModel.find_by_model(:all, :conditions => "parent_models.parent_model_string = 'value'") mods.each {|m| m.class.eql?(GrandchildModel)} # returns true for each model

It is also possible to construct queries that match values for multiple attributes even when the attributes are supported by different models. This is illustrated in the following examples.

# returns a model of class ChildModel mod = ChildModel.find_by_model(:first, :conditions => "parent_models.parent_model_string = 'parent_value' and child_models.child_model_string = 'child_value'") mod.class.eql?(ChildModel) # returns true # returns a model of class GrandchildModel mod = GrandchildModel.find_by_model(:first, :conditions => "parent_models.parent_model_string = 'parent_value' and child_models.child_model_string = 'child_value'") mod.class.eql?(GrandchildModel) # returns true

Note that a query with the :conditions in the example is not possible from ParentModel since ChildModel is not its ancestor. A query for all models that match the specified :condition can be obtained by replacing :first with :all in the example above.

find_by_model also accepts queries for a model by id just like ActiveRecord::Base#find.

Dynamic Finders

Calls to either has_ancestor or has_descendants by a model will add the ability to generate finders to the model. Once the finder is called a new method will be generated and added to the model class. Subsequent calls will use the generated method. The generated finders are used much like the Rails dynamic finders but the generated finders allow queries to be constructed that reference attributes supported by multiple models in the inheritance hierarchy. In the 0.1.2 release the generated finders are limited to conditions that specify equality between an attribute and its value and multiple attribute match conditions may only be aggregated by logical and. More complex queries will require use of finder options. The following examples illustrate use of generated finders by comparison with the equivalent find_by_model call.

mod = ParentModel.find_by_parent_model_string('value') mod = ParentModel.find_by_model(:first, :conditions => "parent_models.parent_model_string = 'value'") mods = ChildModel.find_all_by_parent_model_string('value') mods = ChildModel.find_by_model(:all, :conditions => "parent_models.parent_model_string = 'value'") mod = GrandchildModel.find_by_parent_model_string_and_child_model_string('parent_value', 'child_value') mod = GrandchildModel.find_by_model(:first, :conditions => "parent_models.parent_model_string = 'parent_value' and child_models.child_model_string = 'child_value'") mods = GrandchildModel.find_all_by_parent_model_string_and_child_model_string('parent_value', 'child_value') mods = GrandchildModel.find_by_model(:all, :conditions => "parent_models.parent_model_string = 'parent_value' and child_models.child_model_string = 'child_value'")

For the generated finders to function as expected there can be no name collisions between attributes in the inheritance hierarchy. It is also possible to specify ActiveRecord::Base#find options in generated finders after specification of attribute values. This is illustrated in the following example.

mods = GrandchildModel.find_all_by_parent_model_string('parent_value', :conditions => "child_models.child_model_string = 'child_value'") mods = GrandchildModel.find_by_model(:all, :conditions => "parent_models.parent_model_string = 'parent_value' and child_models.child_model_string = 'child_value'")
